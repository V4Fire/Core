Optional Readonly cacheStrategy

cacheStrategy: CacheStrategy

Inherited from CreateRequestOptions.cacheStrategy

  • Defined in src/core/request/interface.ts:332

Strategy of caching for requests that supports it:

  1. 'forever' - caches all requests and stores their values forever within the active session or until the cache expires (if .cacheTTL is specified);
  2. 'queue' - caches all requests, but more frequent requests will push less frequent requests;
  3. 'never' - never caches any requests;
  4. custom cache object.

Optional middlewares

middlewares: Middlewares<D>

Inherited from CreateRequestOptions.middlewares

  • Defined in src/core/request/interface.ts:372

Dictionary or an iterable value with middleware functions: functions take an environment of request parameters and can modify theirs.

Please notice, that the order of middlewares depends on the structure that you use. Also, if at least one of middlewares returns a function, than the result of invoking this function will be returned as the request result. It can be helpful to organize mocks of data and other similar cases when you don't want to execute a real request.

Optional responseType

responseType: "object" | DataType

Inherited from CreateRequestOptions.responseType

  • Defined in src/core/request/interface.ts:280

Type of the response data: (if not specified, it will be casted dynamically from response headers):

  1. 'text' - result is interpreted as a simple string;
  2. 'json' - result is interpreted as a JSON string;
  3. 'arrayBuffer' - result is interpreted as an array buffer;
  4. 'blob' - result is interpreted as a binary sequence;
  5. 'object' - result is interpreted "as is" without any converting.
